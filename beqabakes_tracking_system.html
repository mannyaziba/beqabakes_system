00What changed

16 problems resolved. Zero formula errors.

The original file was a blank 11-column template that had never been used. The upgraded workbook has 20 fields, corrected dropdowns, live formulas, a Summary dashboard, a Field Guide sheet, and conditional formatting that makes the status of every order visible at a glance.

The most important change: Start using it. A tracking system with zero rows of data is a fiction. From today, every order — however small — goes into the Orders Log before any other action. The Summary sheet only tells you something real when real data is flowing into it.

05Key formulas

527 live formulas. Zero errors. How they work.

All formulas live in the Summary sheet and Column J of the Orders Log. The Orders Log is data entry only — no manual formulas needed there except the Total column which auto-fills. These are the formulas you should understand to trust the numbers.

J2Total (₦) — auto-calculates every row
=IF(G2="","",G2+IF(I2="",0,I2))
If Amount is blank, Total stays blank. If Add-on Amount is blank, it counts as zero. Never shows an error.
B8WA orders this week (Summary)
=IF(B4="","-",COUNTIFS('Orders Log'!A$2:A$501,">="&B4, 'Orders Log'!A$2:A$501,"<="&D4, 'Orders Log'!L$2:L$501,"Paid"))
Counts orders where: Order Date is in the selected week AND Status = Paid. Blank if no week date entered.
B9Revenue this week (₦)
=IF(B4="","-",SUMIFS('Orders Log'!J$2:J$501, 'Orders Log'!A$2:A$501,">="&B4, 'Orders Log'!A$2:A$501,"<="&D4, 'Orders Log'!L$2:L$501,"Delivered"))
Sums Column J (Total ₦) only for Delivered orders in the week. Paid-but-not-yet-delivered orders are excluded from revenue.
B10AOV — average order value
=IF(OR(B4="",B8="",B8=0),"-",B9/B8)
Revenue divided by order count. Shows "-" if either is zero — prevents #DIV/0! error.
B19Conversion rate
=IF(OR(B4="",COUNTIFS('Orders Log'!A$2:A$501,">="&B4, 'Orders Log'!A$2:A$501,"<="&D4)=0),"-", COUNTIFS(…,"Paid")/COUNTIFS(…))
Paid orders as % of all orders in the week (including Pending and Cancelled). Shows the real conversion rate, not just volume.
B33Proof collection rate (all time)
=IF(COUNTIF('Orders Log'!L$2:L$501,"Delivered")=0,"-", COUNTIF('Orders Log'!Q$2:Q$501,"Yes")/ COUNTIF('Orders Log'!L$2:L$501,"Delivered"))
Photos received as % of deliveries. The Proof bank health metric. Target: every delivered order eventually yields a photo — 100% is aspirational, 50%+ is healthy.
If a formula shows an error: check that the sheet name in the formula still reads 'Orders Log' exactly — do not rename Sheet 1. Column references are locked with $. Never insert or delete columns in the Orders Log as this will shift the COUNTIFS column references.
06Weekly workflow

When to update. What to check. Every week.

The discipline that makes this work: The tracking sheet is updated at three moments — when an order is placed, when it is delivered, and every Friday for the review. If these three moments are missed, the data degrades within weeks and the Summary becomes unreliable.
On order Add a new row. Fill Columns A, C, D, E, F, G, H, I, K, L (Pending), M, N (New Lead), S. Leave B (Delivery Date), O, P, Q blank until delivery. Cols A–N, S
On payment Update Column L from Pending → Paid. Update Notion CRM buyer tag simultaneously. Script 01 fires. Col L → Paid
On delivery Fill Column B (Delivery Date). Update Column L → Delivered. Send Script 03 → mark Column O = Yes. Update Buyer Type in Column N if this is a second order (New Lead → Repeat). Cols B, L, N, O
Day 7 Check Notion CRM for buyers with Delivery Date = 7 days ago. Send Script 04. Mark Column P = Yes. If buyer responds and re-orders, upgrade Buyer Type to VIP if this is their 3rd order. Col P → Yes
On proof When buyer shares delivery photo: mark Column Q = Yes. Add to Notion Proof bank with permission noted. ₦500 credit logged in Notes (Column T). Col Q → Yes
Friday review Open Weekly Summary sheet. Enter this week's Monday date in Cell B4. Review all five COS v1 metrics. Check threshold cells. Note any metric below target. Pre-assign next week's three post situations before closing. Summary B4

Notion CRM sync rule

Every time Column N (Buyer Type) is updated in the spreadsheet, make the matching update in Notion CRM. The two systems are not technically linked — they stay in sync through discipline. Notion is the permanent record; the spreadsheet is the operational layer. When they conflict, Notion wins.

New Lead → Repeat

Trigger: second confirmed order in Column L = Paid. Update Column N + Notion buyer type simultaneously.

Repeat → VIP

Trigger: third confirmed order. VIP flag activates loyalty mechanic (Month 3). Update Column N + Notion.

Proof = Yes → Proof bank

Trigger: Column Q = Yes. Photo tagged in Notion Proof bank with date, product, and permission status.

07Expansion thresholds

The Summary sheet tells you when to expand. Watch these four cells.

The four MSOS expansion thresholds all have dedicated cells in the Summary sheet that auto-calculate from the Orders Log. Cells turn green when the condition is met. Nothing is guessed — the spreadsheet makes the decision obvious.

Paid ads gate

15+ WA orders/week

Cell B37 tracks this week's Paid order count. Must hit 15+ for four consecutive weeks before Instagram or Facebook ads activate.

Proof bank target

10+ proof items

Cell B38 counts all-time rows where Column Q = Yes. Must also be 10+ before paid ads activate. Both conditions required simultaneously.

Corporate pipeline

Month 2 activation

Cell B39 counts Gift Set orders (all time). Track corporate interest alongside the Lookbook PDF build and Script 05 readiness.

Loyalty mechanic

5+ VIP buyers

Cell B40 counts VIP entries in Column N. When this hits 5, "Beqa's Circle" stamp-card mechanic activates in Month 3.

The green cell signal: Threshold cells in the Summary sheet are formatted to turn green automatically when their condition is met. On your Friday review, a green threshold cell means one thing: time to act on the next phase. The spreadsheet removes the decision — you just have to look.
